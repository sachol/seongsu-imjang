---
description: 새로운 지역의 임장 대시보드를 생성하는 워크플로우
---

새로운 지역(예: 동탄, 인천 등)의 임장 자료를 바탕으로 대시보드를 생성할 때 다음 단계를 따릅니다.

### 1. 자료 준비 단계

- **필수 데이터:** 지역명, 테마 목록, 테마별 코스 리스트, 체크리스트 항목, 일정 정보.
- **이미지 자료:** 지역 로고, 코스별 이미지 (없을 경우 기본 이미지 사용).

### 2. 데이터 변환 단계

- 사용자가 제공한 텍스트/PDF 초안에서 추출한 데이터를 `src/data/` 폴더 내의 다음 파일 형식으로 변환합니다.
  - `themes.js`: 테마 정보 (ID, 타이틀, 설명, 이미지)
  - `courses.js`: 상세 코스 정보 (테마 ID 매칭, 장소 목록, 특징)
  - `schedule.js`: 임장 스케줄 정보
  - `checklist.js`: 체크리스트 항목

### 3. 브랜딩 및 설정 수정

- `index.html`: `<title>` 태그를 `[지역명] 임장 대시보드`로 변경.
- `.env`: 해당 지역 전용 `VITE_APP_PASSWORD` 설정 확인.

### 4. 검증 및 배포 준비

- `npm run dev`를 통해 로컬에서 데이터가 정상적으로 렌더링되는지 확인.
- `git commit` 및 새로운 GitHub 레포지토리(또는 브랜치)에 푸시.

### 5. Vercel 배포

- Vercel에서 새 프로젝트 연결.
- **Environment Variables**에 `VITE_APP_PASSWORD` 등록.
- 배포 완료 후 최종 URL 확인.
